# test_case

Реализация тестового задания — собственная система аутентификации и авторизации на Django REST Framework.

## 🔐 Аутентификация и роли
- Пользователи регистрируются с указанием имени, фамилии, email и пароля.
- Вход в систему выполняется по email и паролю, используются JWT-токены (access + refresh).
- Роль пользователя хранится в поле `role` модели `User` (значения: `"user"` или `"admin"`).

## 🛡️ Разграничение доступа
- **Гость (неавторизованный):** может только зарегистрироваться или войти.
- **Пользователь (role=user):** может создавать посты, просматривать и редактировать только свои посты.
- **Администратор (role=admin):** имеет доступ ко всем постам (создание, просмотр, редактирование, удаление), может просматривать список всех пользователей и изменять их роли.

Все проверки прав выполняются в коде представлений (View) путём сравнения `request.user.role`.

## 🗑️ Мягкое удаление
Пользователь (или администратор) может деактивировать аккаунт — устанавливается `is_active=False`. После этого даже при наличии действующих токенов доступ к защищённым эндпоинтам блокируется: при попытке обновить токен или войти система возвращает ошибку 401.
